Binary Search Tree (BST)

A Binary Search Tree (BST) is a specialized binary tree that maintains an ordered structure, allowing for efficient searching, insertion, and deletion operations.

Node Structure

  • Key (or Data): The value stored in the node.
  • Left Child: A reference to the left subtree.
  • Right Child: A reference to the right subtree.

BST Properties

  • All keys in the **left subtree** are **less** than the node’s key.
  • All keys in the **right subtree** are **greater** than the node’s key.
  • This ordering allows efficient search and insertion operations.

BST Structure Example

        15
       /  \
      10   20
     / \    \
    8  12   25

In this BST:

  • The **root node** is **15**.
  • All nodes in the **left subtree** of 15 (**10, 8, 12**) have values **less** than 15.
  • All nodes in the **right subtree** of 15 (**20, 25**) have values **greater** than 15.

BST Operations

The following are the key operations performed on a BST:

Insertion

function BST_Insert(root, key):
    if root == null:
        return createNode(key)
    if key < root.key:
        root.left = BST_Insert(root.left, key)
    else:
        root.right = BST_Insert(root.right, key)
    return root

Search

function BST_Search(root, key):
    if root == null or root.key == key:
        return root
    if key < root.key:
        return BST_Search(root.left, key)
    else:
        return BST_Search(root.right, key)

Deletion

function BST_Delete(root, key):
    if root == null:
        return null
    if key < root.key:
        root.left = BST_Delete(root.left, key)
    else if key > root.key:
        root.right = BST_Delete(root.right, key)
    else:

BST Complexity

  • Search: **O(log n)** (Balanced) / **O(n)** (Skewed)
  • Insertion: **O(log n)** (Balanced) / **O(n)** (Skewed)
  • Deletion: **O(log n)** (Balanced) / **O(n)** (Skewed)
