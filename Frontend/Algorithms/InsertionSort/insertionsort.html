Insertion Sort

**Insertion Sort** is a simple yet effective sorting algorithm, commonly used for **small datasets** or when elements are already **partially sorted**. It mimics the way you **sort playing cards in your hand**—by taking one card at a time and placing it in the correct position relative to the others.

How Insertion Sort Works

The algorithm maintains two sections:

  • ✅ **A sorted section** on the left.
  • 📌 **An unsorted section** on the right.

It iterates through the **unsorted section**, picks one element, and **inserts it into its correct position** in the sorted section. This process continues until all elements are in order.

Step-by-Step Example

Let's sort the array [7, 3, 5, 1, 9] using Insertion Sort.

🔹 Step 1: Start with the first element

The first element 7 is already in its correct position.

🔹 Step 2: Insert 3

Compare 3 with 7. Since 3 is smaller, insert it before 7.

[3, 7, 5, 1, 9]

🔹 Step 3: Insert 5

Compare 5 with 7. Insert it before 7, but after 3.

[3, 5, 7, 1, 9]

🔹 Step 4: Insert 1

Compare 1 with all previous elements and insert it at the beginning.

[1, 3, 5, 7, 9]

🔹 Step 5: Insert 9

No changes needed—9 is already in the correct position.

[1, 3, 5, 7, 9]

Insertion Sort Pseudocode

The logic of Insertion Sort can be represented in **pseudocode** as follows: 

function insertionSort(array):
    for i from 1 to length(array) - 1:
        key = array[i]
        j = i - 1
        while j >= 0 and array[j] > key:
            array[j + 1] = array[j]
            j = j - 1
        array[j + 1] = key

Time Complexity Analysis

  • Best-case: O(n) (Already sorted list, only 1 comparison per element).
  • Average-case: O(n²) (Each element may need to shift).
  • Worst-case: O(n²) (Reverse sorted list, most shifting required).

✅ Why Use Insertion Sort?

  • 🔹 **Easy to implement** and understand.
  • 📈 **Efficient for small or nearly sorted datasets**.
  • ⚖ **Stable sorting algorithm** (Preserves order of equal elements).
  • 📝 **Works well for online sorting** (Can sort as new elements arrive).

⚠️ When Not to Use It

  • ❌ **Inefficient for large datasets** (Better alternatives: QuickSort, MergeSort).
  • ❌ **O(n²) complexity makes it slow compared to divide-and-conquer algorithms**.
