Quick Sort

Quick Sort is one of the most **efficient sorting algorithms**. It follows a **divide-and-conquer approach**, meaning it splits the problem into smaller subproblems and solves them recursively.

How Quick Sort Works

The **main idea** behind Quick Sort is selecting a **pivot element** and rearranging the array such that:

  • All elements **smaller** than the pivot go to the left.
  • All elements **greater** than the pivot go to the right.
  • The process **repeats recursively** until the entire array is sorted.

Step-by-Step Example

Let's sort the array [8, 3, 1, 7, 4, 6, 2] using Quick Sort.

Step 1: Choose a Pivot

We select the **last element** 2 as the pivot.

Step 2: Partitioning

Rearrange the array so elements smaller than 2 go to the left:

[1]  [2]  [8, 3, 7, 4, 6]

Step 3: Recursively Sort Left & Right Subarrays

Now we apply Quick Sort to the left and right partitions.

Pseudocode

function quickSort(array, low, high):
    if low < high:
        pivotIndex = partition(array, low, high)
        quickSort(array, low, pivotIndex - 1)
        quickSort(array, pivotIndex + 1, high)

function partition(array, low, high):
    pivot = array[high]
    i = low - 1
    for j from low to high - 1:
        if array[j] < pivot:
            i = i + 1
            swap(array[i], array[j])
    swap(array[i + 1], array[high])
    return i + 1

Time Complexity

  • Best-case: O(n log n) (Balanced partitioning)
  • Average-case: O(n log n)
  • Worst-case: O(n²) (Already sorted array without optimization)

Advantages of Quick Sort

  • **Fast for large datasets** due to **O(n log n)** complexity in most cases.
  • **In-place sorting** (requires minimal extra memory).
  • **Used in real-world applications** like database indexing and system sorting functions.

Disadvantages

  • **Not stable** (relative order of equal elements may change).
  • **Worst-case complexity is O(n²)** (if pivot selection is bad).
